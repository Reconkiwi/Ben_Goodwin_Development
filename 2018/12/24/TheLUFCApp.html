The LUFC News App - Y3S1

The Brief

Having developed the new LUFC News site for Alex, I agreed to start work on a prototype for him for his companies app. Although Alex did not have any designs for the app, he allowed me the freedom to implement my own designs. Therefore, The role was both development and design based.
Alex wanted a simple and streamlined application that effectively pulled data from the site to the app such as posts, users and discussions. Here were Alex’s specifications for the app:

Alex's Specifications

  • Linked to the LUFC News site
  • Consistent design
  • LUFC News branding
  • Look and feel eye-catching
  • Simple to use
  • User logins
  • User chat systems
  • WordPress post data across both platforms
As there were no designs, I was left to build these from my own imagination.

Brainstorming and Wireframing

To ensure that I had a good idea to pitch to Alex during our first meeting, I set out to come up with a series of development tools and wireframes so that we could decide on the best possible solution for development.

Idea 1: Wrapping a Styled HTML Site

My first idea was to create the app as if I were developing a regular HTML website using a ‘one page’ Javascript framework and then subsequently phone wrapping it by placing the whole file inside a blank Ionic src folder and building it from there. This is something that i had done previously for Alex when developing the first prototype for his app.

Problems with Idea 1

My main issue with this idea was that I felt slightly unprofessional and lazy compiling his app in this way. This process works extremely well for very basic prototypes, however, as Alex had already received one of these, I wanted to ensure that his new prototype was to a higher standard than the previous.

Idea 2: Building directly to Ionic using Angular

Another idea that I had was to build the app up from a blank Ionic app. Ionic’s most recent update included Angular integration, therefore, it would be a good chance to learn a new framework and coding language. As well as this Ionic is a popular framework for web app development, so it was likely that there would be plenty of documentation to help me stay on track.

Problems with Idea 2

The issues facing this development were that, as I had suggested earlier, I had never built an application using Ionic nor had I ever used Angular as a coding language. This would mean that it would likely take me longer to learn how to use the language and how the framework works.

Idea 3: Xcode and Android Studio

The final option was to develop the app using Xcode and Android Studio. This would probably be the most professional development outcome as many companies will develop applications directly for each platform IOS and Android whereas frameworks such as Ionic are simple and developed for both platforms simultaneously.

Problems with Idea 3

Unfortunately the largest problem faced with this idea were the time constraints. I did not have enough time to develop an app twice for each platform. As well as this I have never used these technologies before and the amount of work that I would need to put in to get the prototype working was not feasible.

The Final Solution

I eventually decided to pitch the 2nd idea that I had to Alex where I would build an application directly into the Ionic framework. I felt that this would be a good chance for me to learn some new skills and push my development capabilities further.

Development Tools

To gain a better understanding of the development process, I briefly researched into the tools that I would be using for development:

Ionic

Ionic is an open source SDK for rapid hybrid mobile development. This means once the app is created it can be built for any mobile device. This allows for quick creating of applications as well as fast deployment. Ionic 4 now heavily uses Angular for development.

Angular

Angular is a TypeScript open source, front-end web application framework. It is used directly with Ionic 4 and I would need to learn it for the prototypes development.

Rest API's

Rest API’s take advantage HTTP when being used for web, therefore, developers are not required to download libraries or additional software to use them.

  • Problems with the Login Page
The main issue with the login system is that once you are logged in there is no logout function. It keeps the login button available on the navigation menu which directs the user back to login again. Given more time I would have allocated some time to create a session system when if the user is logged in a logout button replaces the login button and by pressing it, it will end the session. As well as this the login system requires a recaptcha to avoid any unwanted spam.
  • The API Problem
Another problem I faced during development was the lack of funding to acquire an API in time for the submission. This meant that I was unable to setup any of the live fixtures, results and league tables on the app as I could not be granted access to the data. This was a bit of a shame but I still mapped out how I felt the pages should look so that Alex could have a good understanding of what he wanted for when he had available funding.
  • The iFlyChat Problem
As I wanted to add a multi-platform chat service to the site and app, I attempted to use a chat plugin for WordPress with multiple platform support. As iFlyChat has HTML integration, I believed that I would be able to call this embed into the index.html of the LUFC News app. Unfortunately, the embed is returning an error which I believe to be caused by the companies outdated code. i have contacted them about this but was unable to receive any information back in time for this submission. This was a massive disappointment as I feel that the chat system would have strengthened the final product much more.
